시안디자인 검토 및 의견 수렴

UI/UX를 고려한 계층구조, 스타일가이드 구성, 시각적 표현 방식에 대해 다른 관계자들의 의견을 적극 수렴하여 디자인을 제작할 수 있다.

시안디자인 및 프로토타입 검증

UX/UI 디자인에서 검증은 특정 시점의 단계적 업무가 아니라 반복적으로 수행하는 활동이 다. 디자인 결과 및 프로토타입 제작 결과에 대한 검증 방법은 전문가들을 통해 이루어지 는 검증을 택하기도 하지만, 대부분 일반 사용자의 시각에서의 검증과 평가를 수행한다.

디자인 선호도(preference) 평가

프로토타입을 통해 확정 시안에 대한 평가를 수행하는 경우, 디자인 선호도 평가를 수행 해 볼 수 있다. 디자인 선호도 평가는 사용자의 감성을 알아보기 위해 과학적인 관점에서 객관적으로 해석하려는 목적을 가진다. 문화콘텐츠 앱의 선호도에 영향을 미치는 속성들 의 중요도를 파악 및 예측하기 위하여 사용되며, 사용자의 감성적 특성을 이해하여 사용 자의 디자인적 니즈를 파악할 수 있다. 평가는 수집하고자 하는 데이터의 특성에 따라 적절한 측정법을 적용해야 하며, 다음과 같은 측정방식을 선택하여 적용해 볼 수있다.

  • 점수(rating)
  • 순위(ordinal ranking)
  • 태도 기반 선호도(attitude based preferences)
  • 속성 기반 선호도(attribute based preferences)
성능(performance) 평가

성능 평가란 사용자가 문화 콘텐츠 앱을 사용해 보고 과업별 학습능력, 효율성, 기억 용이 성, 오류, 만족도 등의 평가 기준에 근거하여 판단하는 것을 말한다. 일반적으로는 학자들 의 UI 원칙, 또는 이러한 원칙들을 수집한 후 병합, 추가, 삭제하는 과정을 거쳐 다음과 같이 자신의 평가 목적에 맞게 평가 기준을 마련하여 사용한다.

  • 단순성 : 정보를 쉽게 찾을 수 있도록 화면 구성이 간결하게 디자인되어 있는지를 묻는다.
  • 일관성 : 각 화면의 디자인은 사용자의 사용 순서에 따라 일관성 있게 배치되어 통일감을 주는 지 아닌지를 묻는다.
  • 가시성 : 화면 정보를 쉽게 찾고 구분할 수 있는지를 묻는다.
  • 효율성 : 효율적으로 작업을 수행할 수 있게 정보와 기능을 사용할 수 있는지를 묻는다.
  • 심미성 : 디자인 요소들이 미적 요소를 충족시키는지를 묻는다.

UX의 검증

UX 평가 모델

UX 평가는 기능적인 측면과 아울러, 사용자의 주관적인 여러 측면의 평가를 알아볼 방법 이다. 피터모빌(Peter Moville)(2004)은 다음과 같은 7가지 측면에서의 UX의 원칙을 수립하 였다. 이를 UX 허니콤(User Experience Honeycomb) 모델이라고 하며, 다음의 7가지 측면 에서 사용자 경험의 수준을 평가하는 것으로 UX를 검증할 수 있다.

  • 유용성 : 사용자의 필요에 따라 편리하게 이용 가능한지 여부를 묻는다.
  • 사용성 : 인터페이스를 사용하기 쉽고 기능에 대한 학습이 용이한지 여부를 묻는다.
  • 매력성 : 감성적 측면에서의 즐거움과 같은 시각적 경험을 제공하는지 여부를 묻는다.
  • 검색성 : 사용자가 찾고자 하는 기능의 발견 여부에 관한 것을 묻는다.
  • 접근성 : 사용자 환경·신체적 기능 제한에 관계없이 누구나 사용 가능한지 여부를 묻는다.
  • 신뢰성 : 제시하는 정보 콘텐츠에 대한 믿음, 에러 여부, 안정성, 반응의 일치 여부를 묻는다.
  • 가치성 : 경험을 통한 사용자의 만족과 서비스 효용 가치 여부를 묻는다.
프로토타입 개선

UX 평가 결과를 분석하여 다른 관계자들과 공유하고, 그들의 의견 수렴을 통해 프로토타 이핑을 이용한 반복적인 분석을 수행한다. 사용자 경험의 수준 평가 항목 중 사용자로부 터 좋지 않은 평가를 받은 항목들에 대해서는 요인 분석을 수행한다. 이를 수정.보완하여 사용자의 니즈에 최대한 부합하는 결과물을 도출하고 프로토타입 평가를 반복 수행한다.
